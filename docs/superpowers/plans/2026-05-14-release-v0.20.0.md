# Release v0.20.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Release Kairox v0.20.0 via PR-based flow — bump versions, update changelog/docs, create PR with auto-merge, tag after merge.

**Architecture:** Follows `docs/releasing.md` PR-based flow. Worktree on branch `chore/release-v0.20.0`, version bump via `just bump-version`, changelog via `git cliff`, PR to main with squash auto-merge, tag push after merge triggers `release-build.yml`.

**Tech Stack:** bash, git, git-cliff, just

### Task 6: Push branch and create PR

- [ ] **Step 1: Push release branch**

```bash
git push -u origin chore/release-v0.20.0
```

- [ ] **Step 2: Create PR with auto-merge enabled**

```bash
gh pr create \
  --title "chore(release): v0.20.0" \
  --body "$(cat <<'EOF'
## Summary
- Bump version to 0.20.0
- Generate CHANGELOG for v0.20.0
- Update ROADMAP with completed items
- Refresh project documentation

## Test plan
- [ ] CI passes all jobs (format, lint, test, type-sync, builds, E2E)
- [ ] `ci-success` aggregation gate passes
- [ ] Auto-merge triggers on gate pass

🤖 Generated with [Claude Code](https://claude.com/claude-code)
EOF
)" \
  --base main
```

- [ ] **Step 3: Enable auto-merge with squash**

```bash
PR_NUM=$(gh pr view --json number -q '.number')
gh pr merge "$PR_NUM" --auto --squash
```

### Task 7: Monitor CI and fix failures

- [ ] **Step 1: Wait for CI jobs to start**

```bash
gh pr checks
```

- [ ] **Step 2: If any check fails, investigate and fix**

Read the failing check logs:

```bash
gh run view <run-id> --log
```

Fix issues in the worktree, commit, push. Repeat until all checks pass and `ci-success` is green.

- [ ] **Step 3: Confirm auto-merge completed**

```bash
gh pr view --json state,mergedAt
```

Expected: `state: MERGED`.
