# Eval Workflow Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce Kairox self-evaluation friction by making message dispatch observable, patch failures easier to repair, pilot startup repeatable, and local evaluation SKILL rules stricter.

**Architecture:** Split into independent lanes with disjoint write scopes. Code PRs stay in tracked Rust/GUI/script files; local `.agents/skills/**` updates are ignored by Git and must be edited in the main checkout without PR. Functional code lanes use TDD and keep Dev App verification explicit where user-visible behavior is affected.

**Tech Stack:** Rust, Tauri 2 commands, Vue generated command bindings, agent-tools patch parser, Bun scripts, local Kairox SKILL markdown.
