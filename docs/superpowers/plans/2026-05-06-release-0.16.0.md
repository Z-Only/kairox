# Release v0.16.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cut Kairox release **v0.16.0** by bumping every version-of-truth file, refreshing project + AI docs, regenerating CHANGELOG, opening a PR with auto-merge, and pushing the release tag after merge to trigger the auto-build pipeline.

**Architecture:** Release work happens in an isolated git worktree (`.worktrees/chore-release-0.16.0`). Version bumps use the existing `just bump-version` recipe. CHANGELOG is generated by `git-cliff` (already configured via `cliff.toml`). Tagging/publishing is post-merge so that the tag points at the merged commit and `release-build.yml` picks it up.

**Tech Stack:** git + git worktree, `just`, `git-cliff` 2.13.1, `gh` CLI (GitHub auth: `Z-Only`), pnpm, cargo.
