# GUI UI Primitives Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the audited Kairox GUI interaction issues while introducing a small AI-friendly headless primitives layer backed by `reka-ui`.

**Architecture:** Keep Kairox's existing CSS token visual system and add wrapper components under `apps/agent-gui/src/components/ui/`. Business UI should use Kairox wrappers, not raw third-party primitives. Behavior changes are implemented test-first across Vitest, Playwright mock/E2E, and tauri-pilot scenarios.

**Tech Stack:** Vue 3 Composition API, TypeScript, Pinia setup stores, Tauri 2 IPC, Rust runtime facade, Vitest, Playwright, tauri-pilot, CSS custom properties, `reka-ui` headless primitives.
