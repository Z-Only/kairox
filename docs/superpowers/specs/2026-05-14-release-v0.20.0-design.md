# Release v0.20.0 — Design

## Goal

Release Kairox v0.20.0 via PR-based flow. Bump from 0.19.0 → 0.20.0.

## Scope

Minor version bump. Key changes since v0.19.0:

- feat(gui): comprehensive UI polish — color, typography, accessibility, micro-interactions (#155)
- chore(deps): upgrade Rust and JS dependencies (#156)

## Approach

PR-based flow as documented in `docs/releasing.md`. NOT direct-push via `scripts/release.sh`.

## Steps

1. Create worktree on branch `chore/release-v0.20.0`
2. `just bump-version 0.20.0` — updates Cargo.toml, Cargo.lock, package.json (×2), tauri.conf.json
3. `git cliff --tag v0.20.0 -o CHANGELOG.md` — auto-generate changelog from conventional commits
4. Update ROADMAP.md — mark completed items
5. Review and update AGENTS.md, CLAUDE.md, README for stale references
6. Review banner.svg for needed updates
7. Commit, push, create PR targeting main
8. Enable auto-merge (squash)
9. Monitor CI, fix failures
10. After merge: `git pull --ff-only origin main`, tag `v0.20.0`, push tag

## CI gate

- `ci.yml` on PR: format, clippy, web lint, cargo test, type-sync, TUI build, GUI build, Playwright E2E, tauri-pilot E2E, live model smoke
- `ci-success` aggregation job must pass for auto-merge
- Release Build (`release-build.yml`) triggers on `v0.20.0` tag push
